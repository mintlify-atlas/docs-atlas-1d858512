## Overview

The `Logo` component displays the DriveLoop app logo with customizable dimensions. It centers the image and handles both numeric and string-based sizing.

## Import

```typescript
import Logo from '../components/Logo';
```

## Props

The component extends all `ViewProps` from React Native, plus the following:

<ParamField path="width" type="number | string" default="192">
  The width of the logo. Can be a number (pixels) or a Tailwind class string
</ParamField>

<ParamField path="height" type="number | string" default="96">
  The height of the logo. Can be a number (pixels) or a Tailwind class string
</ParamField>

<ParamField path="style" type="StyleProp<ViewStyle>">
  Additional styles for the container View
</ParamField>

<ParamField path="className" type="string">
  Tailwind CSS classes for additional styling (spacing, etc.)
</ParamField>

## TypeScript Interface

```typescript
import { ViewProps } from 'react-native';

interface LogoProps extends ViewProps {
    width?: number | string;
    height?: number | string;
}
```

## Features

- **Flexible Sizing**: Accepts both numeric (pixels) and string (Tailwind) values
- **Centered Layout**: Logo is automatically centered in its container
- **Aspect Ratio**: Uses `resizeMode="contain"` to preserve aspect ratio
- **Asset Management**: Loads from local assets directory

## Component Source

Location: `~/workspace/source/components/Logo.tsx:9-23`

```typescript
const Logo = ({ width = 192, height = 96, style, ...props }: LogoProps) => {
    return (
        <View className="items-center" style={style} {...props}>
            <Image
                source={require('../assets/images/logo.png')}
                style={{
                    width: typeof width === 'number' ? width : undefined,
                    height: typeof height === 'number' ? height : undefined
                }}
                className={typeof width === 'string' ? width : ''}
                resizeMode="contain"
            />
        </View>
    );
};
```

<Note>
  The component intelligently handles both numeric and string width/height values. Numeric values set inline styles (pixels), while string values are passed as className for Tailwind utilities.
</Note>

<Tip>
  The `resizeMode="contain"` ensures your logo maintains its aspect ratio and fits within the specified dimensions without distortion.
</Tip>

## Sizing Strategies

### Fixed Pixel Sizes
Best for:
- Consistent branding across screens
- Precise design requirements
- Icons and small logos

```typescript
<Logo width={128} height={64} />
```

### Responsive with Tailwind
Best for:
- Adaptive layouts
- Responsive designs
- Dynamic content

```typescript
<Logo width="w-48" height="h-24" />
```

## Asset Requirements

The logo asset should be located at:
```
~/workspace/source/assets/images/logo.png
```

For best results:
- Use PNG format with transparency
- Provide @2x and @3x versions for retina displays
- Maintain a 2:1 aspect ratio (width:height)
- Optimize file size for mobile

## Performance Tips

</Tip>

  If you need to load logos dynamically from a URL, modify the component to accept a `source` prop and use `{ uri: 'https://...' }` format.
</Note>
