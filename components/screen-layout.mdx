## Overview

The `ScreenLayout` component is a wrapper for screen content that handles safe areas (notches, status bars), provides optional scrolling, and maintains consistent padding across the app.

## Import

```typescript
import ScreenLayout from '../components/ScreenLayout';
```

## Props

<ParamField path="children" type="React.ReactNode" required>
  The content to render inside the screen layout
</ParamField>

<ParamField path="scrollable" type="boolean" default="true">
  If true, wraps content in a ScrollView. If false, uses a static View
</ParamField>

<ParamField path="paddingHorizontal" type="number" default="6">
  Horizontal padding units (uses Tailwind's spacing scale)
</ParamField>

## TypeScript Interface

```typescript
interface ScreenLayoutProps {
    children: React.ReactNode;
    scrollable?: boolean;
    paddingHorizontal?: number;
}
```

## Features

- **Safe Area Management**: Uses `SafeAreaView` to respect device notches and system UI
- **Flexible Scrolling**: Choose between scrollable and fixed layouts
- **Consistent Padding**: Configurable horizontal padding
- **White Background**: Clean white background by default
- **Full Height**: Takes full screen height with `flex-1`

## Component Source

Location: `~/workspace/source/components/ScreenLayout.tsx:11-24`

```typescript
const ScreenLayout = ({ children, scrollable = true, paddingHorizontal = 6 }: ScreenLayoutProps) => {
    const Container = scrollable ? ScrollView : View;

    return (
        <SafeAreaView className="flex-1 bg-white">
            <Container
                contentContainerStyle={scrollable ? { flexGrow: 1 } : undefined}
                className={`flex-1 px-${paddingHorizontal}`}
            >
                {children}
            </Container>
        </SafeAreaView>
    );
};
```

<Note>
  The component dynamically chooses between `ScrollView` and `View` based on the `scrollable` prop, providing flexibility without code duplication.
</Note>

<Tip>
  Use `scrollable={false}` for screens with fixed layouts like splash screens, error pages, or centered content that shouldn't scroll.
</Tip>

## When to Use Each Mode

### Scrollable (Default)
Use for:
- Forms with multiple inputs
- Content that may exceed screen height
- Lists or feeds
- Article/document viewing
- Any dynamic content

### Non-Scrollable
Use for:
- Splash screens
- Empty states
- Error pages
- Fixed dashboards
- Camera/AR views
- Content that should always fit on screen

## Padding Scale

Tailwind spacing units (based on 4px):
- `0` = 0px
- `2` = 8px
- `4` = 16px (common for cards/grids)
- `6` = 24px (default, good for forms)
- `8` = 32px (extra spacious)

## Safe Area Behavior

The component uses `react-native-safe-area-context` to handle:
- iPhone notches (Dynamic Island)
- Android status bars
- Navigation bars
- Home indicators
- Rounded screen corners

<Note>
  Ensure `SafeAreaProvider` wraps your app root (typically in `_layout.tsx`) for safe areas to work properly.
</Note>

## Keyboard Handling

For forms, consider wrapping with `KeyboardAvoidingView`:

```typescript
import { KeyboardAvoidingView, Platform } from 'react-native';
import ScreenLayout from '../components/ScreenLayout';

const FormScreen = () => {
    return (
        <KeyboardAvoidingView 
            behavior={Platform.OS === 'ios' ? 'padding' : 'height'}
            className="flex-1"
        >
            <ScreenLayout>
                {/* Form content */}
            </ScreenLayout>
        </KeyboardAvoidingView>
    );
};
```
